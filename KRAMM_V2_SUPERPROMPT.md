## SECTION 3: QUESTION EVALUATOR AGENT (NEW — CRITICAL)

**File to Create:** backend/ai/question_evaluator.py

This agent sits BETWEEN the quiz request and the quiz generator. Its sole responsibility is to evaluate and curate which questions to use for a specific quiz.

### ⚠️ BANK-FIRST MANDATE (NON-NEGOTIABLE)

**Questions are chosen MOSTLY from the pre-built question banks (NCERT textbook bank + Novel CBSE-style bank).** New questions are generated by the quiz generator ONLY in these two cases:

1. **Exhaustion**: All relevant questions in both banks for the given topic have been exhausted (i.e., the student has already seen every available question on this topic).
2. **Language/Angle Mismatch**: The student's answer history shows a pattern of confusion or wrong answers that suggests the existing bank questions' language, phrasing, or angle isn't connecting — the student needs the same concept tested from a fundamentally different perspective that doesn't exist in the banks.

**The LLM decision loop for question selection must be in sync with student history and progress.** The evaluator agent must understand the RELATIONAL understanding between topics and question formats — i.e., which question types/formats/angles work best for which topics given this specific student's learning pattern. For example: if a student consistently fails SA questions on "Quadratic Equations" but passes MCQs, the evaluator should recognize this pattern and either (a) select more SA questions to strengthen that weakness, or (b) if bank SA questions are exhausted, request generation of new SA questions specifically targeting the gap.

**The objective is singular: prepare Ridham for the CBSE Class 10 board exam.** Every question selection decision must serve this goal.

### THREE QUIZ TYPES

#### Type 1: TOPIC QUIZ (Homework-Based)
**When:** Parent scans homework and selects "Quiz Mode"
**Objective:** Test understanding of the specific topic just studied
**Strategy:** 100% from question banks FIRST. Generate ONLY if banks exhausted or language mismatch detected.
**Difficulty:** Start at student's current mastery level, escalate per question
**Duration:** 6-8 questions, 15-20 minutes

Selection algorithm:
1. Load question_bank_index to find keyword matches
2. Load matching chapter from question_bank/ AND novel_question_bank/
3. Filter questions: topic_relevant AND not_previously_seen AND difficulty_appropriate
4. Analyze student's answer_history for this topic — check if specific question formats cause consistent failure (relational topic-format understanding)
5. Sort by: Bloom's level (respect progression), CBSE_weightage (high first), novelty (haven't seen before), format-weakness targeting
6. Select top 6-8 from banks
7. ONLY IF < 6 quality candidates remain in BOTH banks → generate remainder via quiz_generator
8. ONLY IF student's answer patterns show language/angle confusion → generate re-angled questions

#### Type 2: CHAPTER QUIZ (One-Click Per Chapter)
**When:** Triggered from "Quick Quiz" button on any chapter/topic in dashboard
**Objective:** Broad assessment of chapter mastery
**Strategy:** 100% from question banks FIRST. Generate ONLY if banks exhausted per sub-topic or language mismatch.
**Requirement:** At least 1 question from each sub-topic in the chapter
**Mix:** ~50% MCQ + ~50% subjective
**Duration:** 8-10 questions, 25-30 minutes

Selection algorithm:
1. Map chapter_number → sub_topics array (from ncert_chapters JSONs)
2. For each sub_topic: select 1 best unseen question from banks (NCERT first, then Novel)
3. Check student's format-weakness patterns per sub-topic — prefer formats student struggles with
4. ONLY IF a sub_topic has zero unseen bank questions → generate for that sub-topic
5. Ensure difficulty distribution: 30% easy, 40% medium, 30% hard

#### Type 3: WEAK AREA REVISION QUIZ (Cross-Topic)
**When:** Triggered from "Revision Quiz" button (new)
**Objective:** Spaced repetition on weak topics using research-based recall intervals
**Strategy:** 100% from banks FIRST (prioritize previously-failed question TYPES). Generate ONLY if all bank questions on weak concepts are exhausted, OR if student's answer history shows the existing question angles aren't working (language/perspective mismatch).
**Requirement:** Must include ≥1 question re-testing each weak concept from a DIFFERENT angle than before
**Duration:** 5-8 questions, variable

Selection algorithm:
1. Call student_memory.get_weak_areas() → get list of {topic, concepts, fail_count, last_failed_date}
2. For each weak area: pull 1-2 UNSEEN questions from banks that re-test same concept
3. Analyze student's answer history on these concepts — identify which question formats/angles consistently fail
4. Select bank questions that approach the concept from a different format than previously failed ones
5. ONLY IF no unseen bank questions exist for a weak concept → generate new questions on same concepts but different question types/angles
6. Prioritize: longest_time_since_last_attempt first (spaced repetition)
7. If concept hasn't been retested in 7+ days → higher priority
8. Build quiz spanning 3-5 topics (not just one topic)

### EVALUATION CRITERIA (for each candidate question)

Score each question on these dimensions (0-100 per dimension):

1. **Topic Relevance** (0-100)
   - Exact match to quiz topic: +100
   - Related concept in topic: +70
   - Same chapter: +30
   - Different chapter: 0

2. **Difficulty Appropriateness** (0-100)
   - Based on student's mastery_score for this topic:
     - Mastery < 30%: easy questions only (easy=100, medium=50, hard=0)
     - Mastery 30-60%: easy & medium (easy=80, medium=100, hard=50)
     - Mastery > 60%: all levels (easy=60, medium=80, hard=100)

3. **Bloom's Level Progression** (0-100)
   - Student has only mastered "remember": only ask "remember" Q (100), "understand" (50), others (0)
   - Student mastered "apply": can ask up to "apply" (100), "analyze" (50), others (0)
   - Student mastered "analyze": can ask any level (100)

4. **CBSE Exam Weightage** (0-100)
   - High-weightage topics: +100
   - Medium-weightage: +70
   - Low-weightage: +40

5. **Novelty Check** (0-100)
   - Question NOT in quiz_log.questions_seen: +100
   - Question seen once before (>7 days ago): +70
   - Question seen recently (<7 days): 0

6. **Concept Coverage** (0-100)
   - Does this question test a concept the student hasn't seen yet: +100
   - Tests a concept they got wrong before: +100
   - Tests a concept they know: +50

7. **Type Balance** (0-100)
   - If quiz needs more MCQ: MCQ=100, others=50
   - If quiz needs more subjective: SA/LA=100, MCQ=50

**Scoring Formula:**
```
final_score = (
  topic_relevance * 0.20 +
  difficulty_appropriateness * 0.25 +
  bloom_progression * 0.15 +
  cbse_weightage * 0.15 +
  novelty * 0.10 +
  concept_coverage * 0.10 +
  type_balance * 0.05
) / 100
```

### EVALUATOR FLOW (BANK-FIRST)

```
evaluate_and_select(quiz_request) →
  1. Load question_bank_index.json + novel_question_bank_index.json
  2. Match topic keywords → identify relevant chapter numbers
  3. Load matching chapter JSONs from both banks into memory
  4. Load student_context from student_memory
  5. Load quiz_history to get questions_seen_ids
  6. Load answer_history to analyze topic-format relationships (relational understanding)
  7. Create candidate_pool = all questions from loaded banks
  8. FOR EACH candidate question:
       - Filter OUT if: already_seen OR off_topic
       - Score on 7 dimensions + topic-format-weakness bonus
       - Calculate final_score
  9. Sort candidates by final_score (descending)
  10. Select top N candidates (where N = target_question_count)
  11. CHECK: Are there enough bank candidates to fill the quiz?
       - YES → Use 100% bank questions. Done.
       - NO (banks exhausted) → Generate ONLY the remaining count via quiz_generator
       - SPECIAL CASE: If student's answer_history shows repeated confusion/wrong answers
         with specific language/angles on this topic → Generate re-angled questions
         EVEN IF bank questions exist (but still prefer banks for majority)
  12. Randomize order of selected questions
  13. Return quiz object with full question_objects + source attribution
  14. Log decision rationale: {bank_count, generated_count, generation_reason}
```

## SECTION 9: MOM-FIRST REVEAL FLOW (NEW — CRITICAL)

### THE FLOW

This is the complete answer → evaluation → reveal flow:

```
1. STUDENT ANSWERS (via Telegram text, voice, or image)
   ↓
2. ANSWER EVALUATOR AGENT runs (backend)
   - Evaluates answer against CBSE standard
   - Generates feedback, score, conceptual gap
   - Generates avatar emotion for teaching agent
   ↓
3. EVALUATION SENT TO MOM ON TELEGRAM FIRST
   - Mom sees: question, student's answer, correct answer, evaluation, score
   - Mom sees: "Reveal on TV" button
   - TV dashboard shows: LOCKED state (answer submitted, waiting for Mom)
   - Teaching agent on TV: shows "thinking" emotion
   ↓
4. MOM TAPS "REVEAL ON TV" BUTTON
   - Telegram sends callback to backend
   - Backend emits Socket.io event: "quiz:reveal_answer"
   ↓
5. TV DASHBOARD UNLOCKS THE ANSWER
   - Answer card transitions from locked → revealed
   - Correct answer displayed
   - Evaluation feedback displayed
   - Score updated
   - Teaching agent shows emotion + speaks feedback message
   ↓
6. ANSWER SAVED TO HISTORY
   - answer_history.json updated
   - Student can now see the full evaluation on TV
   ↓
7. NEXT QUESTION (or quiz complete)
```

```python
async def handle_reveal_callback(update, context):
    """Callback handler for 'reveal_answer|quiz_id|question_index'"""

        student_context=student_context
    )
    await sio.emit("teaching:message", teaching_response)
    
    # Confirm to Mom
    await query.answer("✅ Revealed on TV!")
    
    # Save to answer history
    student_memory.save_answer(evaluation)
```

```python
        # Already have subject, chapter, topic from context
```

```jsx
function RadarChartPanel({ data }) {
  const skillNames = [
    'Quantitative',
    'Analytical',
    'Logical\nReasoning',
    'Conceptual\nUnderstanding',
    'Scientific\nReasoning',
    'Procedural\nFluency',
    'Problem\nSolving'
  ];
  
  const radarData = skillNames.map((name, i) => ({
    subject: name,
    score: Object.values(data.current_scores)[i]
  }));
  
  return (
    <div className="radar-panel">
      <h3>Cognitive Skills</h3>
      <ResponsiveContainer width="100%" height={400}>
        <RadarChart data={radarData}>
          <PolarGrid stroke="rgba(147, 51, 234, 0.3)" />
          <PolarAngleAxis dataKey="subject" stroke="#D1D5DB" />
          <PolarRadiusAxis angle={90} domain={[0, 100]} stroke="#9CA3AF" />
          <Radar 
            name="Score" 
            dataKey="score" 
            stroke="var(--accent-purple)" 
            fill="var(--accent-purple)" 
            fillOpacity={0.3}
          />
        </RadarChart>
      </ResponsiveContainer>
      
      <div className="skill-legend">
        {Object.entries(data.current_scores).map(([skill, score]) => (
          <div key={skill} className="skill-item">
            <span className="skill-name">{formatSkillName(skill)}</span>
            <span className="skill-score">{score}</span>
          </div>
        ))}
      </div>
    </div>
  );
}

function formatSkillName(skill) {
  return skill
    .split('_')
    .map(word => word.charAt(0).toUpperCase() + word.slice(1))
    .join(' ');
}
```

```jsx
              <div 
                key={topic.name} 
                className={`topic-card strength-${getStrengthLevel(topic.mastery_score)}`}
              >
                <div className="topic-header">
                  <span className="topic-name">{topic.name}</span>
                  <span className={`trend-arrow ${topic.trend}`}>
                    {topic.trend === 'improving' ? '↗' : topic.trend === 'declining' ? '↘' : '→'}
                  </span>
                </div>
                <div className="topic-score">{topic.mastery_score}%</div>
                <div className="topic-meta">
                  <span className="last-tested">{formatDate(topic.last_quiz_date)}</span>
                </div>
              </div>
            ))}
          </div>
        </div>
      ))}
    </div>
  );
}

function getStrengthLevel(score) {
  if (score >= 70) return 'high';
  if (score >= 40) return 'medium';
  return 'low';
}
```

```jsx
function QuizDetailModal({ quizId, onClose }) {
  const [quiz, setQuiz] = useState(null);
  const [currentQuestionIndex, setCurrentQuestionIndex] = useState(0);
  
  useEffect(() => {
    fetch(`/api/analytics/quiz-detail/${quizId}`)
      .then(r => r.json())
      .then(data => setQuiz(data));
  }, [quizId]);
  
  if (!quiz) return <div className="modal-loading">Loading...</div>;
  
  const current = quiz.questions[currentQuestionIndex];
  const isCorrect = current.student_answer === current.correct_answer;
  
  return (
    <div className="quiz-detail-modal-overlay">
      <div className="quiz-detail-modal">
        <div className="modal-header">
          <h2>{quiz.topic}</h2>
          <button onClick={onClose} className="close-btn">✕</button>
        </div>
        
        <div className="quiz-progress">
          <div className="progress-bar">
            <div className="progress-fill" style={{width: `${(currentQuestionIndex / quiz.questions.length) * 100}%`}} />
          </div>
          <span className="progress-text">{currentQuestionIndex + 1} / {quiz.questions.length}</span>
        </div>
        
        <div className="question-detail">
          <div className={`answer-status ${isCorrect ? 'correct' : 'incorrect'}`}>
            {isCorrect ? '✓ Correct' : '✗ Incorrect'}
          </div>
          
          <h3 className="question-text">{current.question_text}</h3>
          
          <div className="answer-section">
            <div className="student-answer">
              <h4>Your Answer</h4>
              <div className={`answer-box ${isCorrect ? 'correct' : 'incorrect'}`}>
                {current.student_answer}
              </div>
            </div>
            
            {!isCorrect && (
              <div className="correct-answer">
                <h4>Correct Answer</h4>
                <div className="answer-box correct">
                  {current.correct_answer}
                </div>
              </div>
            )}
          </div>
          
          <div className="feedback-section">
            <h4>📝 Feedback</h4>
            <p className="feedback-text">{current.feedback}</p>
            <p className="mom-explanation">{current.mom_explanation}</p>
          </div>
          
          <div className="question-metadata">
            <span className="metadata-item">
              <strong>Difficulty:</strong> {current.difficulty}
            </span>
            <span className="metadata-item">
              <strong>Time:</strong> {current.time_taken_seconds}s
            </span>
            <span className="metadata-item">
              <strong>Marks:</strong> {current.marks}
            </span>
          </div>
        </div>
        
        <div className="modal-navigation">
          <button 
            onClick={() => setCurrentQuestionIndex(Math.max(0, currentQuestionIndex - 1))}
            disabled={currentQuestionIndex === 0}
            className="nav-btn"
          >
            ← Previous
          </button>

            onClick={() => setCurrentQuestionIndex(Math.min(quiz.questions.length - 1, currentQuestionIndex + 1))}
            disabled={currentQuestionIndex === quiz.questions.length - 1}
            className="nav-btn"
          >
            Next →
          </button>
        </div>
      </div>
    </div>
  );
}
```

### Endpoint 1: GET /api/analytics/curriculum-timeline

```python
@app.get("/api/analytics/curriculum-timeline")
async def get_curriculum_timeline():
    """
    Returns per-subject chapter progress timeline.
    Merges data from:
    - NCERT chapters list (ch count, names)
    - child_profile.json (topic_history, quiz_history)
    - student_memory.json (weak_areas)
    """
    try:
        # Load NCERT chapters metadata
        with open("data/ncert_chapters_math.json") as f:
            math_chapters_meta = json.load(f)
        with open("data/ncert_chapters_science.json") as f:
            science_chapters_meta = json.load(f)
        
        # Load child profile
        with open("data/child_profile.json") as f:
            child_profile = json.load(f)
        
        # Load student memory
        student_memory = StudentMemory(Path("data"))
        topic_mastery = student_memory.get_skill_profile()  # Get all topics
        
        result = {"math": [], "science": []}
        
        # Process Math chapters
        for chapter_meta in math_chapters_meta:
            chapter_num = chapter_meta["chapter_number"]
            
            # Find matching topic_history entries
            topics_covered = [
                {
                    "name": topic["name"],
                    "date": topic["studied_date"]
                }
                for topic in child_profile.get("topic_history", [])
                if topic["chapter"] == chapter_num and topic["subject"] == "math"
            ]
            
            # Find matching quiz_history entries
            quizzes = [
                {
                    "quiz_id": quiz["id"],
                    "date": quiz["created_at"],
                    "score": quiz["score_percentage"]
                }
                for quiz in child_profile.get("quiz_history", [])
                if quiz.get("chapter") == chapter_num and quiz.get("subject") == "math"
            ]
            
            # Determine status
            if quizzes:
                status = "tested_in_quiz"
            elif topics_covered:
                status = "covered_in_class"
            else:
                status = "not_started"
            
            # Check for coverage gaps
            coverage_gaps = [
                topic["name"]
                for topic in topics_covered
                if not any(q["quiz_id"] in [quiz["id"] for quiz in child_profile.get("quiz_history", [])] for q in quizzes)
            ]
            
            result["math"].append({
                "chapter_number": chapter_num,
                "name": chapter_meta["name"],
                "status": status,
                "topics_covered": topics_covered,
                "quizzes": quizzes,
                "coverage_gaps": coverage_gaps
            })
        
        # Repeat for Science chapters
        for chapter_meta in science_chapters_meta:
            chapter_num = chapter_meta["chapter_number"]
            
            topics_covered = [
                {

            ]

                    "date": quiz["created_at"],
                    "score": quiz["score_percentage"]
                }
                for quiz in child_profile.get("quiz_history", [])
                if quiz.get("chapter") == chapter_num and quiz.get("subject") == "science"
            ]
            
            if quizzes:
                status = "tested_in_quiz"
            elif topics_covered:
                status = "covered_in_class"
            else:
                status = "not_started"

                for topic in topics_covered
                if not any(q["quiz_id"] in [quiz["id"] for quiz in child_profile.get("quiz_history", [])] for q in quizzes)
            ]

                "name": chapter_meta["name"],
                "status": status,
                "topics_covered": topics_covered,
                "quizzes": quizzes,
```
